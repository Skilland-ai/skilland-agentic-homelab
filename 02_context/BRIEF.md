# BRIEF

## What This Project Is
- `skilland-agentic-homelab` is the experimental, breakable HomeLab for Skilland/Reboot on the real VPS.
- Its purpose is to turn the strategic North Star into practical learning about agentic operations.
- HomeLab starts as a product/capability discovery lab, not as a closed technical architecture, client product, or enterprise security model.

## Who It Is For
- Primary operator and decision owner: Raúl.
- Primary resident technical operator: Codex, working inside the repo and VPS.
- Future operators: Hermes and other resident AI agents with roles, memory, tools, and traceability.
- Downstream beneficiaries: Skilland/Reboot operations, Divi-DClick where learnings are safe and sellable, and future productized agentic OS patterns.

## Core Outcome
- Convert the VPS from loose infrastructure into an understandable agentic lab where Codex, Hermes, agents, repos, and simple human command channels can be explored through short microhitos.

## Initial Focus
- First strong microhito: Hermes Deep Research / Hermes Literacy.
- Second microhito: Hermes Intimacy through installation, configuration, breakage, logs, memory, agents, skills, and runtime observation.
- Later work: Codex-Hermes relationship, first real repo operation, human channels, and multi-repo/meta-harness learning.

## Measurable Outcome
- Context layer is initialized and usable.
- Hermes sources are gathered and ranked before installation.
- The real VPS/runtime state is inspected and documented.
- Each microhito leaves a lightweight run note with objective, actions, learnings, failures, provisional decisions, open questions, and next step.
- At least one real repo is eventually operated for useful work, not as a demo.

## Time Horizon
- Immediate: initialize context and prepare Hermes Deep Research.
- Near term: research Hermes deeply, then touch/install/break it on the VPS.
- Medium term: clarify Hermes-Codex roles and operate a first real repo.
- Long term: derive safe patterns for Divi-DClick, productization, and an internal Agentic Company OS.

## Success Definition
- Codex understands the HomeLab mission and can continue work inside the VPS without re-reading raw inbox.
- Hermes moves from chosen-by-intention to understood-by-practice.
- Raúl can return to the VPS, see state, decisions, and next steps clearly, and keep momentum.
- Repos start being treated as operable factories/departments.
- The project learns quickly, documents well, and avoids premature architecture.

## Unknowns
- Unknown - authoritative Hermes documentation and community sources are not yet collected.
- Unknown - actual Hermes installation status on this VPS.
- Unknown - exact Node/npm/runtime state in the current environment.
- Unknown - which real repo should be operated first after Hermes research.
- Unknown - exact boundaries for infrastructure changes beyond the broad approval protocol.
